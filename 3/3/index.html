    3. declaration and assignment

    1. 类型和声明

    对类型和声明的翻译过程中，需要：

    • 类型检查: 保证运算数类型和运算符匹配，例如 Java 要求 && 只能作用于 boolean
    • 存储分配: 根据声明的类型确定存储空间大小，在符号表中添加变量名对应的类型和相对偏移量
    • 其他: 例如计算数组引用指示的地址、显式类型转换等等

    (1). 类型表达式

    类型表达式由基本类型和类型构造算子组成。举例来说 (用 T 表示类型表达式)：

    • 基本类型: 如 void、char 等
    • 类名
    • 数组构造符: array (integer, T)
    • 指针构造符: pointer (T)
    • 笛卡尔乘积构造符: T1 × T2 × ...
    • 函数构造符: 例如 T1 × T2 × ... × Tn → R
    • 记录构造符: 例如 record ((N1 × T1) × (N2 × T2) × ... (Nn × Tn))，其中 Ni 表示标识符

    (2). 局部变量的存储分配

    有了类型表达式，就可以确定标识符的类型。

    e.g.3-1 一个实现了存储分配的 SDT 
    ① P → { offset = 0 } D
② D → T id ; {enter (id.lexeme, T.type, offset); offset = offset + T.width; } D
③ D → ε
④ T → B { t = B.type; w = B.width; } C { T.type = C.type; T.width = C.width;}
⑤ T → ↑ T1 { T.type = pointer (T1.type); T.width = 4 }
⑥ B → int { B.type = int; B.width = 4; }
⑦ B → real { B.type = real; B.width = 8; }
⑧ C → ε { C.type = t; C.width = w; }
⑨ C → [num] C1 { C.type = array (num.val, C1.type); C.width = num.val * C1.width; }
    • $\small\rm D$ 负责生成声明的序列，$\small\rm T$ 负责生成类型，包含基本类型、数组类型和指针类型
    • $\small\rm B$ 产生具体的基本类型，$\small\rm C$ 产生 0 个或多个用方括号括起的整数
    • 产生式 ④ 引入了两个局部变量 $\small\it t$ 和 $\small\it w$，也就是 $\small\rm C$ 的继承属性，用于在 $\small\mathrm{C}\to\varepsilon$ 时为 $\small\rm T$ 计算最终的属性值

    2. 表达式

    (1). 简单赋值语句的翻译

    e.g.3-2 简单赋值语句的 L-SDT 
    ① S → id = E; { p = lookup(id.lexeme); if p == nil then error ; gen(p = E.addr); } 
② E → E1 + E2 { E.addr = newtemp(); gen(E.addr = E1.addr + E2.addr); }
③ E → E1 ∗ E2 { E.addr = newtemp(); gen(E.addr = E1.addr ∗ E2.addr); }
④ E → - E1 { E.addr = newtemp(); gen(E.addr = unminus E1.addr; }
⑤ E → ( E1 ) { E.addr = E1.addr; }
⑥ E → id { E.addr = lookup(id.lexeme); if E.addr == nil then error }
    • E.addr : 表示存放表达式 $\small\rm E$ 的值的地址 (三地址指令的"地址"，即变量名)
    • lookup(name) : 查询符号表，返回 name 对应的记录
    • newtemp() : 生成一个新的临时变量 t，返回它的地址
    • gen(code) : 增量添加一个三地址指令

    (2). 数组引用的翻译

    对于一个 $\small k$ 维数组的某个元素 $\small a[i_1][i_2]...[i_k]$，其相对地址为 $\small\mathit{base}+i_1\times w_1+...+i_k\times w_k$，其中 $\small w_j$ 是 $\small a[i_1][i_2]...[i_j]$ 指代的元素的宽度。

    e.g.3-3 与例子 3-2 的 SDT 组合起来，形成完整的赋值文法 
    ⑦ S → L = E; { gen(L.array[L.offset] = E.addr); } 
⑧ E → L { E.addr = newtemp(); gen(E.addr = L.array[L.offset]); }
⑨ L → id [E] { L.array = lookup(id.lexeme); if L.array == nil then error; L.type = L.array.type.eleme;
         L.offset = newtemp(); gen(L.offset = E.addr ∗ L.type.width) }
⑩ L → L1 [E] { L.array = L1.array; L.type = L1.type.eleme; t = newtemp();
         gen(E.addr ∗ L.type.width); L.offset = newtemp(); gen(L.offset = L1.offset + t); }
    • L.array : 查询符号表所得的地址；L.array.type 是其类型表达式，eleme 给出了其类型，即 a$\small[i_1]$ 的类型
    • L.offset : 起到临时变量的作用，用于累加 $\small i_j\times w_j$
    • newtemp() : 生成一个新的临时变量 t，返回它的地址

    设 type(a)=array(3, array(5, array(8, int))), 那么 addr($\small a[i_1][i_2][i_3]$) = base + $\small i_1\cdot 160+i_2\cdot 32+i_3\cdot 4$。若使用上述 SDT 翻译，得到的三地址码序列为
     $\small t_1=i_1*160$, $\small t_2=i_2*32$, $\small t_3 = t_1+t_2$, $\small t_4=i_3*4$, $\small t_5=t_3+t_4$, $\small t_6=a[t_5]$
