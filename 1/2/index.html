    2. some details

    1. 冲突

    当输入集合的多个前缀与一个或多个模式匹配时，就会造成冲突。

    (1). 最长前缀

    可以通过如下方式解决冲突：

    • 总是选择最长的前缀；
    • 如果最长的前缀与多个模式匹配，选择最先列出的模式。
    第一个规则使得 ++、<= 这样的运算符可以被正确识别，第二个规则使得关键字称为保留字符。

    (2). 向前看运算符

    有些时候，我们希望某个词素后面跟随某个特定的字符时，这个词素才与某个特定的模式相匹配。

    在 Fortran 语言中，关键字并不是保留字。在 IF(I, J)=3 中，IF 是一个数组名；在
    IF (condition) THEN... 中，IF 是一个关键字。

    可以在关键字后添加一个附加的模式，例如 IF /(.*) {letter}。

    2. 词法分析器的结构

    (1). NFA 的模拟

    NFA 的模拟类似于边构造边使用的子集构造法，算法中使用两个栈和一个查询数组。假设 NFA 有 $n$ 个点、$m$ 条转移以及串的长度为 $\small k$，那么算法花费的时间为 $\small O(k(m+n))$。

    Transit(oldStates, newStates, c):
1   for (s : oldStates) {
2       for (t : move(s, c))
3           if (! on[t])
4               Add-State(t, newStates)
5       pop(s, oldStates)
6   }
7   for (s : newStates) {
8       pop(s, newStates)
9       push(s, oldStates)
10      on[s] = FALSE
11  }

    NFA-Simulate():
1   oldStates = ε-closure(s0)
2   c = nextChar()
3   while (c != eof) {
4       Transit(oldStates, newStates, c)
5       c = nextChar()
5   }
7   if (oldStates ∩ F != ∅) return "yes"
8   else return "no"

Add-State(s, newStates):
1   push(s, newStates)
2   on[s] = TRUE
3   for (t : move(s, ε)) 
5       if (! on[t])
6           Add-State(t)














    (2). 基于 NFA 的词法分析器

     首先把每个正则表达式转换为一个 NFA，然后通过空转移和一个新的开始状态把这些 NFA 合并为一个。合并的时候要考虑各个模式的顺序，以处理冲突。
     在这个 NFA 模拟运行的过程中，会终止在一个没有任何后续状态的输入点，此时就可以判定匹配，且如果匹配则一定是最长前缀。

    (3). 基于 DFA 的词法分析器

     把合并后的 NFA 通过子集构造法转化为 DFA，再模拟它的执行。

    3. 基于 DFA 的模式匹配器的优化

    (1). DFA 最小化：Hopcroft 算法

    由 等价状态 的定义可知，DFA 最小化的过程就是把状态集划分为若干等价类的过程。则 Hopcroft 算法的思路如下：

    定义: 设 $S$ 的一个划分为 $P=\lbrace B_1,...,B_n\rbrace$，如果 $P$ 满足
    ① 任何 $B_i$ 中的所有状态要么是接受态，要么是非接受态
    ② 如果状态 $s$ 与 $t$ 等价，则它们属于同一个 $B_i$
    就称 $P$ 是一个合法的划分。
    引理1: 设 $P$ 是一个合法的划分。若 $\exists s,t\in B_i, a\in\Sigma$，使得 $\delta(s, a)\in B_j, \delta(t, a)\notin B_j$，就可以在 $P$ 中把 $B_i$ 进一步划分为
      $B_i'=\lbrace s\in B_i\ |\ \delta(s,a)\in B_j\rbrace$ 和 $B_i''=\lbrace s\in B_i\ |\ \delta(s,a)\notin B_j\rbrace$
    且新划分 $P'$ 仍是一个合法的划分。

    $\small\bullet$ Hopcroft 算法的框架就是不断选取集合作为 distinguisher 划分初始集合，直到不可划分为止，此时每个集合一定都是一个等价类。但盲目选取 distinguisher 难以保证效率。

    引理2: 如果划分 $P$ 中所有 $S$ 的子集都已经用 $(B_j, a)$ 检查过得到了 $P'$，那么以后不再需要通过 $(B_j, a)$ 检查了。

    $\small\bullet$ $P'$ 中任一子集中的元素要么都在 $B_j$ 中，要么都不在，且此后的划分也满足这一点。

    引理3: 设某个 $B_i$ 被分裂为 $B_i'$ 和 $B_i''$，则对任意 $a\in\Sigma$，从 $\lbrace(B_i,a),(B_i',a),(B_i'',a)\rbrace$ 中任选两个检查当前划分 $P$，与使用全部三个是等效的。

    $\small\bullet$ 使用 $B_i'$ 和 $B_i''$ 检查后，对任意子集 $A\in P''$，以及 $\forall s\in A,a\in\Sigma$，有以下可能：
     ① $\small\delta(s,a)\in B_i',\delta(s,a)\in B_i''$    ② $\small\delta(s,a)\notin B_i',\delta(s,a)\in B_i''$
     ③ $\small\delta(s,a)\in B_i',\delta(s,a)\notin B_i''$    ④ $\small\delta(s,a)\notin B_i',\delta(s,a)\notin B_i''$
    因此，已没必要再用 $\small(S_i,a)$ 检查。

    DFA-Minimization():
    P = {F, Q - F }, W = {F }
    while (W != ∅) {
        A = W.pop()
        for (c : Σ) {
            X = {q | δ(q, c) ∈ A, q ∈ S }
            for (Y in P and X ∩ Y is nonempty and Y - X is nonempty) {
                replace Y in P by X ∩ Y and Y - X
                if (Y is in W)
                    replace Y in W by the same two sets
                else
                    if (|X ∩ Y| <= |Y - X|) add X ∩ Y to W
                    else                          add Y - X to W
            }
        }
    }















    伪代码如上所示，其中 W 是 distinguishers 的集合。

    引理4: 对 $\forall B\in P$ 和 $a\in\Sigma$，如果 $\small (B,a)\notin W$，则永远也不需要 $(B,a)$ 去检查。

    $\small\bullet$ 由引理 3，初始情况下此结论成立，并且在 $\small W$ 不断弹出元素的过程中，此结论也成立。因此，当 $\small W$ 为空时，算法结束。

    算法复杂度：$O(n\lg n)$
